#### 2.6 订单状态机与校验（前后端一致）

**允许的状态迁移矩阵**：

| From | To | 条件/动作 |
|------|----|-----------|
| `pending` | `paid` | 支付成功（H5 Mock 或真实支付回调） |
| `paid` | `confirmed` | 管理端确认订单基础信息 |
| `confirmed` | `assigned` | 分配陪诊员（需 `escortId`） |
| `assigned` | `in_progress` | 陪诊员到达并开始服务 |
| `in_progress` | `completed` | 服务完成、生成服务报告 |
| 任意 | `cancelled` | 条件取消（未开始或平台退单规则） |
| `paid`/`confirmed` | `refunded` | 合规退款通道（支付通道回调） |

**后端示例校验（伪代码）**：

```ts
// server/src/modules/orders/orders.service.ts
// 说明：通过条件更新（updateMany）保证原子性，避免并发状态错乱
async function updateStatus(id: string, from: string, to: string) {
  const allowed = new Set([
    'pending:paid', 'paid:confirmed', 'confirmed:assigned',
    'assigned:in_progress', 'in_progress:completed',
    'any:cancelled', 'paid:refunded', 'confirmed:refunded',
  ])
  const key = `${from}:${to}`
  if (!allowed.has(key) && to !== 'cancelled') throw new Error('非法状态迁移')

  // 使用条件更新保证并发安全（受影响行数为 1 才视为成功）
  const { count } = await this.prisma.order.updateMany({
    where: { id, status: from },
    data: { status: to },
  })
  if (count !== 1) throw new Error('状态已变化或订单不存在')
}
```

```ts
// server/src/modules/escort/escort-orders.service.ts
// 说明：抢单使用事务 + 条件更新，确保只有一个陪诊员能成功
async function grabOrder(orderId: string, escortId: string) {
  return this.prisma.$transaction(async (tx) => {
    // 原子抢占：只有 status 为 paid 的订单可以被抢占为 assigned
    const { count } = await tx.order.updateMany({
      where: { id: orderId, status: 'paid' },
      data: { status: 'assigned', escortId },
    })
    if (count !== 1) throw new Error('订单已被抢走或不可抢')
    return { success: true }
  })
}
```

**数据库建议**：

- 为 `order.status` 建立索引，提高条件更新性能
- 保留审计字段（如 `assignedAt`、`assignedBy`）便于追踪
- 若业务允许，增加“陪诊员-订单唯一约束”以避免二次分配

## 🛡️ 避坑指南 (审计补充)

> 基于 Taro 跨端开发实战经验，以下是 5 个**必须提前知道的坑**。

### 坑 1: 地图组件的"隐形大坑" 🗺️

**问题**: `<Map />` 组件在小程序端是原生组件，性能极好；但在 H5 端需要依赖第三方地图 SDK。

**现象**: H5 开发时地图一片空白。

**解决方案**:

```typescript
// MVP 建议：H5 阶段用静态图片代替
// miniapp/src/components/MapView/index.tsx

import { isH5 } from '@/utils/env-adapter'
import { Map, Image } from '@tarojs/components'

export const MapView = ({ latitude, longitude, markers }) => {
  if (isH5) {
    // H5 使用静态图 (高德静态地图 API)
    const staticMapUrl = `https://restapi.amap.com/v3/staticmap?location=${longitude},${latitude}&zoom=15&size=750*300&markers=mid,,A:${longitude},${latitude}&key=YOUR_AMAP_KEY`
    return <Image src={staticMapUrl} mode="widthFix" />
  }
  
  // 小程序使用原生 Map
  return <Map latitude={latitude} longitude={longitude} markers={markers} />
}
```

### 坑 2: 手机号授权的"环境差异" 📱

**问题**: 小程序端用 `<Button open-type="getPhoneNumber">`，H5 里这个按钮没反应。

**解决方案**: 创建跨端手机号组件

```typescript
// miniapp/src/components/PhoneAuth/index.tsx

import { isH5 } from '@/utils/env-adapter'
import { Button, Input, View } from '@tarojs/components'
import { useState } from 'react'

interface Props {
  onGetPhone: (phone: string) => void
}

export const PhoneAuth = ({ onGetPhone }: Props) => {
  const [inputPhone, setInputPhone] = useState('')

  // H5 模式：显示输入框
  if (isH5) {
    return (
      <View className="phone-auth-h5">
        <Input 
          placeholder="输入手机号" 
          value={inputPhone}
          onInput={(e) => setInputPhone(e.detail.value)}
        />
        <Button onClick={() => onGetPhone(inputPhone || '13800138000')}>
          确认 (H5 模拟)
        </Button>
      </View>
    )
  }

  // 小程序模式：微信授权按钮
  return (
    <Button 
      openType="getPhoneNumber"
      onGetPhoneNumber={(e) => {
        if (e.detail.code) {
          // 调用后端解密获取手机号
          onGetPhone(e.detail.code)
        }
      }}
    >
      微信一键授权
    </Button>
  )
}
```

```typescript
    cancelText: '拒绝',
    success: (res) => {
      resolve({ event: 'agree', buttonId: res.confirm ? 'agree-btn' : '' })
    },
  })
})
```

### 坑 5: 样式单位 (rpx vs px) 的"消失边框" 📏

**问题**: `1px` 的边框在某些手机上会消失（被四舍五入）。

**解决方案**:

```scss
// 方案 1: 使用大写 PX (不被转换)
.card {
  border: 1PX solid #eee; // 大写 PX 不会被 Taro 转换
}

// 方案 2: 使用 hairline 伪元素
.card::after {
  content: '';
  position: absolute;
  left: 0;
  bottom: 0;
  width: 100%;
  height: 1px;
  background: #eee;
  transform: scaleY(0.5);
}
```
