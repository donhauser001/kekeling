### 1.6 数据覆盖机制（P0 页面）

#### 类型定义

```typescript
// types.ts
export interface MarketingDataOverride {
  // 会员信息（null 表示未开通，undefined 不覆盖）
  membership?: MembershipInfoOverride | null
  // 会员套餐列表
  membershipPlans?: MembershipPlanOverride[]
  // 我的优惠券
  coupons?: { items?: CouponItemOverride[]; total?: number }
  // 可领取优惠券
  availableCoupons?: AvailableCouponOverride[]
}
```

#### 使用示例

```tsx
// 管理后台页面中即时预览
<TerminalPreview
  page="membership"
  marketingData={{
    membership: {
      levelName: '黄金会员',
      expireAt: '2025-12-31',
      points: 1280,
    },
    membershipPlans: [
      { id: '1', name: '月度会员', price: 29, isRecommended: true },
    ],
  }}
/>

// 模拟未开通会员
<TerminalPreview
  page="membership"
  marketingData={{ membership: null }}
/>
```

#### 覆盖规则

| 字段值 | 行为 |
|--------|------|
| `undefined` | 不覆盖，使用 API 数据 |
| `null` | 覆盖为空（如未开通会员） |
| `object/array` | 使用覆盖数据，不请求 API |

> ⚠️ 覆盖数据使用 `Partial` 类型，无需提供完整结构，组件内部会补充默认值

分销功能主要通过以下方式体现：
1. **邀请好友页面**（`/pages/marketing/referrals/index`）- 已归类到营销中心
2. **个人中心入口** - 分销关系查看（如存在）
3. **后台管理** - 分销配置、分润记录查看

### 3.2 建议

- **优先级：P2（低）**
- 先完成营销中心和陪诊员系统集成
