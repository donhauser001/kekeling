# 文件结构

## 模块目录结构

每个列表页模块应采用以下文件结构：

```
src/features/{module-name}/
├── index.tsx                      # 主页面（数据获取、弹窗状态、布局）
└── components/
    ├── {module}-columns.tsx       # 列定义
    ├── {module}-table.tsx         # 表格组件
    ├── {module}-action-dialog.tsx # 新建/编辑弹窗
    ├── {module}-detail-sheet.tsx  # 查看详情抽屉（可选）
    └── {module}-delete-dialog.tsx # 删除确认弹窗（可选，可复用 ConfirmDialog）
```

---

## 命名规范

### 文件命名

使用 `kebab-case`（短横线连接）：

```
campaigns-columns.tsx
campaigns-table.tsx
campaigns-action-dialog.tsx
```

### 组件命名

使用 `PascalCase`（大驼峰）：

```tsx
export function CampaignsTable() { ... }
export function CampaignsActionDialog() { ... }
```

### 列定义函数

使用 `get{Module}Columns` 格式：

```tsx
export function getCampaignsColumns({ onView, onEdit, onDelete }: ColumnsProps) {
  return [...]
}
```

---

## 文件职责

| 文件 | 职责 |
|------|------|
| `index.tsx` | 数据获取（useQuery）、弹窗状态管理、页面布局 |
| `{module}-table.tsx` | 表格配置（useReactTable）、渲染、工具栏、翻页 |
| `{module}-columns.tsx` | 列定义、操作列菜单 |
| `{module}-action-dialog.tsx` | 新建/编辑表单弹窗 |
| `{module}-detail-sheet.tsx` | 详情查看抽屉 |

---

## 示例：员工列表模块

```
src/features/hr/
├── index.tsx
└── components/
    ├── employees-columns.tsx
    ├── employees-table.tsx
    ├── employees-action-dialog.tsx
    ├── employees-delete-dialog.tsx
    └── employees-invite-dialog.tsx
```
